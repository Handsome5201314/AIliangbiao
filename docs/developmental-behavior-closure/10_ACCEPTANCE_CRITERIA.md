# 10 验收标准

## 总体验收

- 不重写项目。
- 不破坏现有核心功能。
- 保留 Next.js + TypeScript + Prisma + PostgreSQL。
- 七个目标量表可查询。
- 量表最终计分由确定性代码或 MCP API 完成。
- 医生复核为正式报告强制前置。
- AI 调用可审计。
- 科研导出默认脱敏。
- 所有量表必须记录授权状态，商业化前必须授权核验。

## 医生端验收

- 医生必须登录且审核通过。
- 医生可创建或选择儿童档案。
- 医生可发起目标量表评估。
- 医生可查看待复核队列。
- 医生复核后才能生成正式报告。
- 医生端可下载/打印已复核报告。
- 医生复核操作有审计日志。

## 家长端验收

- 家长可通过账号密码登录。
- 医生邀请链接可免登录填写。
- 免登录链接不可查看正式报告。
- 低置信度答案映射需要追问确认。
- `physician_review` 量表提交后显示等待医生复核。
- 复核后报告查看需要登录或二次校验。

## 科研端验收

- 研究导出不含直接身份字段。
- 历史导入有批次、字段映射和质量标记。
- 研究衍生表字段与数据字典一致。
- 3 个月 75-105 天窗口判断正确。
- 导出行为记录操作者、用途、格式和字段清单。

## 比赛端验收

- Skill 与 MCP 职责边界清楚。
- demo 模式只使用模拟或脱敏数据。
- MCP 负责确定性计分。
- Skill 输出不得包含诊断、处方或绕过医生复核的正式报告。
- 调用链可复现、可审计。
