# 06 工作流

## 医生端门诊评估

1. 医生登录并通过审核。
2. 创建或选择儿童档案。
3. 选择首批目标量表之一。
4. 发起门诊评估，可选择医生现场录入或生成家长 H5/Handoff 链接。
5. 系统保存 `AssessmentSession`。
6. 用户完成量表后，确定性计分写入 `AssessmentHistory`。
7. 系统创建或展示待复核项。
8. 医生查看原始答案、总分、维度分、AI 辅助摘要和注意事项。
9. 医生完成复核。
10. 系统生成正式报告并允许医生下载/打印。
11. 医生创建 1 个月/3 个月复测任务。

## 家长 H5 居家复测

1. 家长账号密码登录，或通过医生邀请链接进入填写。
2. 免登录邀请链接只允许填写，不允许查看正式报告。
3. 家长逐题作答，可使用 AI 解释和追问确认。
4. 低置信度答案必须确认后保存。
5. 提交后由确定性引擎计分。
6. `physician_review` 模式下，提交端只显示“等待医生复核”。
7. 正式报告必须医生复核后，通过登录或二次校验才可查看。

## 科研端

1. 导入历史 CSV。
2. 配置字段映射和缺失值规则。
3. 生成脱敏研究编号。
4. 合并系统干预组自动沉淀数据。
5. 计算主结局和次要结局。
6. 导出脱敏研究衍生表。

Phase 0 已确认当前仓库有脱敏导出基础，但历史导入批次、字段映射和质量标记仍需新增。

## 健康教育和随访

1. 医生复核完成。
2. 系统读取量表类型、风险等级、维度分和问题域。
3. 匹配人工审核健康教育内容。
4. 推送给家长端或在报告后展示。
5. 记录打开、阅读、确认。
6. 创建 1 个月/3 个月复测任务。
7. 医生或护士手工提醒。
8. 复测完成后进入趋势分析和科研导出。

AI 只能解释和匹配已审核内容，不能临时编造医学健康教育。

## Skill + MCP 比赛演示

1. 外部智能体调用 Skill。
2. Skill 根据任务调用 MCP 工具。
3. MCP 创建评估会话、获取题目、提交答案、确定性计分。
4. Skill 生成家长解释或医生摘要草稿。
5. 安全审查后输出演示结果。
6. 比赛模式使用模拟或脱敏数据，必须带 `DEMO_ONLY` 语义或水印。
