# 06 工作流

本文描述当前代码已经支持或已经进入模型/API/服务层的核心业务路径。页面体验仍会持续打磨，但工作流边界以这里为准。

## 医生端门诊评估

1. 医生注册、登录，并通过管理员审核。
2. 医生创建或选择儿童档案，也可以通过门诊二维码或邀填链路沉淀档案。
3. 医生选择目标量表，发起门诊评估、医生现场录入或家长 H5/Handoff 填写。
4. 系统保存 `AssessmentSession`，并通过统一量表目录读取题目、选项和交互模式。
5. 家长或医生完成答案后，系统调用确定性计分引擎，不信任客户端提交的总分。
6. 计分结果写入 `AssessmentHistory`、`ScaleScore` 和 `resultDetails`。
7. 医生复核流程创建或读取 `DoctorReview`。
8. 医生完成复核：通过、退回、要求补充信息或拒绝。
9. 通过复核后生成 `AssessmentReport`。
10. 家长端只能查看已复核报告。

## 门诊二维码手机筛查

1. 医生或后台创建门诊筛查点和量表二维码。
2. 家长扫码进入 `/clinic/qr/[slug]`。
3. 页面采集必要身份信息。
4. 进入手机版一题一页答题体验。
5. 题目解释通过底部弹出层展示，不遮挡答题。
6. 提交走门诊 QR 专用公开接口，不要求患者登录。
7. 提交成功后显示筛查编号，并进入医生复核流程。

## 家长 H5 / Handoff

1. 家长账号密码登录，或通过医生邀请链接进入填写。
2. 免登录邀请链接只允许填写和提交，不允许查看正式报告。
3. H5 从后端读取真实会话、题目和当前进度，不使用本地 mock 数据作为正式来源。
4. 家长逐题作答；AI 只用于题目解释。
5. `physician_review` 模式下，提交后只显示“等待医生复核”。
6. 正式报告必须医生复核后，通过登录或二次校验才可查看。

## AI 解释与语音

1. ASR/TTS Provider 在后台配置。
2. 题目解释调用项目自己的解释 API，再由该 API 使用后台配置的 AI Provider 或知识库服务。
3. AI 输出只用于解释、引导和辅助理解，不直接提交答案。
4. 模糊回答不自动猜测；页面应要求用户选择明确选项或补充说明。
5. `AiConversationSession` / `AiConversationEvent` 记录 ASR、确认、fallback、TTS、tool call 和最终答案提交。
6. `/admin/ai-logs` 是正式复盘入口；科研导出从项目数据库读取并默认脱敏。

## 科研端

1. 历史 CSV 通过科研导入 API 进入 `ResearchImportBatch`。
2. 字段映射写入 `ResearchFieldMapping`，行级数据写入 `ResearchImportRow`。
3. 缺失值、质量标记和导入错误保留在批次与行级记录中，不把缺失分数强行记为 0。
4. 系统干预组数据从评估、复核、报告查看、健康教育、随访和 AI 交互日志中自动沉淀。
5. 科研导出默认脱敏，生成 HMAC 研究编号，不导出姓名、电话、门诊号、住院号等直接标识符。
6. `ResearchDerivedDataset` 保存主结局、次要结局、质量摘要、行快照和内容 hash。

## Skill + MCP

1. 外部智能体调用 Skill 或 MCP。
2. Skill 使用用户态 `agent session`，MCP 使用系统态 API Key。
3. Skill/MCP 可查询量表目录、创建会话、提交答案、映射自然语言、确认低置信度答案、确定性计分和查询结果。
4. 演示脚本使用 `DEMO_ONLY` 合成数据，不放宽生产授权，也不绕过医生复核边界。
5. Canonical MCP 工具调用写入 `McpToolLog`，legacy 兼容入口保留 `McpLog`。
6. Skill 可以生成家长解释或医生摘要草稿，但正式报告和科研导出仍以项目数据库和复核流程为准。
