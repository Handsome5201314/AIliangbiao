# 09 分阶段路线图

## Phase 0 仓库审计与 Project Harness

只审计和落库文档，不改业务代码。输出本目录 00-14 文档，明确方向、边界、数据模型、接口契约、AI 安全、医生复核和后续开发路线。

## Phase 1 数据模型与 Prisma 扩展

新增或扩展医生复核、正式报告、报告模板、健康教育、随访任务、提醒日志、科研导入、量表授权、AI/MCP 日志模型。必须包含 Prisma migration、schema 测试和回滚说明。

## Phase 2 量表引擎与 MCP API

统一量表目录和确定性计分 API，收敛旧 `/api/assessment/save`，补齐自然语言映射确认、低置信度追问和 MCP/Skill 工具契约。

## Phase 3 医生端门诊评估

医生创建儿童档案、选择量表、发起评估、生成 H5 链接、查看进度和待复核队列。优先完成门诊主流程，不做营销式页面。

## Phase 4 家长 H5 居家复测

支持账号密码登录、邀请链接填写、断点续填、AI 逐题解释和低置信度确认。免登录链接只能填写，不允许查看正式报告。

## Phase 5 正式报告模板与 PDF/打印

落地 `DoctorReview` 和 `AssessmentReport` 状态机，支持 SNAP-IV、ABC、通用报告模板，生成 PDF/打印版。报告必须医生复核后可见。

## Phase 6 健康教育与随访闭环

建设人工审核健康教育内容库、触达日志、阅读记录、1 个月/3 个月复测任务和手工提醒记录。

## Phase 7 科研导入导出和研究衍生表

支持历史 CSV 导入、字段映射、质量标记、脱敏研究编号、衍生表版本、主结局/次要结局导出。

## Phase 8 Skill + MCP 比赛演示包

准备 5 个模拟或脱敏案例，配置 `demo_mode`，展示 Skill + MCP 调用链、确定性计分、医生复核边界和安全审查。

## Phase 9 科室内部试用与授权核验

完成科室内部试用、日志审计、权限复核、授权状态核验、隐私检查、部署 smoke check 和回滚预案。商业化前不得把未授权量表标记为可商用。
