# 11 Project Harness 工程护栏

## 本项目硬边界

1. 基于现有仓库继续开发，不重写技术栈。
2. 保留 Next.js + TypeScript + Prisma + PostgreSQL。
3. 第一批量表只聚焦 M-CHAT-R、ABC、CARS、SRS、SNAP-IV、ATEC、VINELAND-3。
4. AI 可以用于逐题对话采集、题目解释、自然语言答案映射、低置信度追问确认、家长版解释、医生复核摘要、健康教育内容匹配和解释、比赛端 Skill 演示。
5. AI 不允许用于量表最终计分、最终风险等级判定、诊断结论、药物或治疗处方、正式报告审批、绕过医生复核、科研导出字段规则自由生成。
6. 所有量表计分必须由确定性代码或 MCP API 完成。
7. 所有正式报告必须医生复核后可见。
8. 医生复核为必选流程。
9. 家长免登录 H5 链接只允许填写，不允许查看正式报告。
10. 查看正式报告需要登录或二次校验。
11. 初期不接真实短信。
12. 健康教育内容必须人工审核，AI 不得临时编造医学内容。
13. 报告模板必须支持医院/科室抬头、LOGO、医生签名可配置。
14. 默认报告抬头为：解放军总医院第一医学中心。
15. 当前阶段为科室内部试用和科研验证，所有量表必须记录 `license_status`，商业化前必须授权核验。
16. 不允许把真实儿童隐私数据写入测试代码。
17. 不允许无 migration 修改数据库。
18. 不允许删除现有核心功能。
19. 不允许大规模重构，除非文档中明确列出风险、范围和回滚方案。
20. 每个阶段必须有验收标准和测试建议。

## 开发顺序

每个后续 Phase 默认顺序：

1. 读取本目录文档。
2. 读取 `README.md`、`prisma/schema.prisma`、目标模块和相关测试。
3. 明确 Source of Truth、Module 边界、接口契约、数据流和风险。
4. 小步实现，不做范围外重构。
5. 运行匹配风险的验证。
6. 更新本目录或 Trellis/spec 中的长期规则。

## 数据库护栏

- 任何 Prisma schema 变更必须有 migration。
- 不允许在生产路径里做自动迁移。
- 不允许用默认值伪装未知授权、未知诊断或未知报告状态。
- 不允许把旧字段和新字段长期并行作为兼容层，迁移窗口必须有删除条件。

## 权限护栏

- 医生端写操作必须验证医生身份和患者访问权限。
- 家长端只能访问本人或授权成员数据。
- MCP API Key 和 agent session token 不可混用。
- 外部智能体不得越权查看正式报告或真实儿童隐私数据。

## 当前 Phase 0 说明

当前仓库未见 `.trellis`。本轮用户明确限定主要修改范围为 `docs/developmental-behavior-closure/` 和必要 README 入口，因此未初始化 Trellis。后续若进入多阶段多人/多 Agent 并行开发，建议单独执行 Trellis 初始化和 spec 同步任务。
